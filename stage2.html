3 - Notions abordées pendant le stage

a - JQuery

Jquery est une bibliothèque gratuite et libre qui est compatible avec la majorité des navigateurs. Le framework permet de simplifier l'écriture du code en JS. JQuery permet d'interagir avec les éléments du DOM (Document Object Model), la gestion des evenements (clics, double clics, au survol, scroll, ...) et bien d'autres choses ...
Par example :
$(document).ready(function () { $('button').on('click', function () { alert('test') }); });
Ce code permet d'afficher une alert si on clique sur un élément enfant de la balise body, aprés le chargement de la page.

b - SweetAlert

SweetAlert est un plugin qui permet d'afficher des alertes (Popup) stylisées. SweetAlert posséde un large panel de paramétres qui permet de modifier le comportement des alertes (icon, title, text, showConfirmButton, ...)
Par example :
Swal.fire( title: 'Good job!', text: 'You clicked the button!', icon : 'success' )
Ce code permet d'afficher une alerte avec un icon succés, "Good job!" comme titre et "You clicked the button!" comme texte de l'alerte.

c - AJAX

L' AJAX (Asynchronous JavaScript and XML) est une technologie qui permet de rendre une page web interactive. C'est-à-dire qu'il permet de communiquer avec un serveur à l'aide de code javascript qui s'exécute en arrière-plan. L'AJAX permet donc d'envoyer des requêtes asynchrones vers un serveur tout en continuant à afficher du contenu à l'écran sans attendre forcément une réponse de cette requête.
Ainsi l'AJAX est très utile lorsque l'on veut mettre à jour un formulaire par exemple sans devoir pour autant actualiser la page. De plus les requêtes AJAX peuvent renvoyer des donées formatées en XML ou en JSON.

d - Le JSON

Le JSON (JavaScript Objet Notation) est un langage qui permet de représenter des données structurées. C'est un langage aussi simple à comprendre pour l'ordinateur qu'un humain.
Par example :
{"employees":[ { "prenom":"John", "nom":"Doe" }, { "prenom":"Anna", "nom":"Smith" }, { "prenom":"Peter", "nom":"Jones" } ]}
Ce code représente le nom et prénom de tous les employés d'une entreprise par exemple. Ainsi avec une boucle je peux afficher l'ensemble des employés !

e - L' API Dolibarr

Une API (Interface de Programmation d'Application) permet de simplifier le développement de notre application / interface web. En effet grâce à une API, il est possible d'envoyer des requêtes pour récupérer des données de l'API et les afficher sur notre application / interface web. Ainsi cela simplifie et rend plus rapide le développement de certaines applications.

f - Les tâches CRON

Une tâche CRON est un programme qui automatise les tâches à exécuter à des moments prédéterminés.
Un développeur peut configurer une CRON pour qu'elle exécute des tâches à intervalles réguliers, comme une fois par jour ou une fois par semaine. Les CRON peuvent également être configurés pour exécuter des tâches à des moments précis, comme à une heure précise chaque jour ou à une date spécifique.

Lors de mon stage les CRON étaient utilisées pour exécuter un script php qui exécute une requête SQL qui permettait de déterminer si un utilisateur avait confirmé une commande (statutCommande = 1 ), si c'était le cas un script pour envoyer le récapitulatif de la commande par mail est exécuté. Ainsi l'utilisateur et le producteur et le clients qui a passé la commande reçoivent tous deux un mail avec la récapitulatif de la commande.
